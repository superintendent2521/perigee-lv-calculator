Mission Page · Layout Reorganizations
Four ways to re-arrange the existing mission elements — mission picker, launch setup (LV / payload / orbit), vehicle roster & multi-vehicle ops, the events list + add-event, the band / node-map view, the state monitor, the ΔV budget, the orbit catalog, PNG export — into something more intuitive. No new copy telling you what to do; the structure does the guiding. Same dark / mono style.
A

Plan / Fly

Two modes, one toggle

The split: today everything is on screen at once. This splits the page into two modes via the top toggle. PLAN = set up the rocket and build the event list (no big chart competing for space). FLY = the chart goes full-width with the scrubber, state monitor and roster. The single [ Plan · Fly ] toggle is the only new affordance and it reads instantly: "set it up" vs "watch it run." Each mode is half as busy as today.

B

Inspector

Select → edit on the right

The split: a left browser (the mission's vehicles & events as a tree), a big center view, and a single right inspector that always shows whatever is selected — click an event and its editor appears; click a vehicle and its multi-vehicle ops appear; click empty space and you get add-event. Replaces today's scattered side panels with one predictable rule: select something → it's editable on the right. The roster/setup/event-editor stop fighting for the same column.

C

Stages

Setup · Flight Plan · Analysis

The split: three sub-tabs inside the mission, mirroring the natural order of work. Setup (vehicle, payload, orbit), Flight Plan (roster + events + add-event), Analysis (band/node view + state monitor + ΔV budget + PNG). Same tab pattern you already use for Spacecraft/Fleet/Missions, so it's instantly familiar — and each tab is calm instead of the current everything-at-once wall.

D

Flight Strip

View on top, timeline below

The split: keep one page but stack it by purpose. The view (band/node map) spans the top as the hero. Directly beneath runs the event timeline as a horizontal strip aligned under it, with the active-vehicle roster as a compact left dock and an inline ＋ between events for inserting. The ΔV budget + state ride a thin bottom status bar. Vertical order = "see it → plan it → check it," and the horizontal event strip lines up with the chart so cause and effect are visually obvious.
